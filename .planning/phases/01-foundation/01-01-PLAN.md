<task type="auto">
  <name>Task 2: Configure Tailwind v4 + shadcn/ui</name>
  <files>
    - package.json
    - electron.vite.config.ts
    - components.json
    - src/renderer/src/assets/main.css
    - src/renderer/src/lib/utils.ts
  </files>
  <action>
    1. Install Tailwind v4:
       `npm install -D tailwindcss @tailwindcss/vite`

    2. Add @tailwindcss/vite plugin to electron.vite.config.ts renderer section:
       ```typescript
       import tailwindcss from '@tailwindcss/vite'
       // in renderer.plugins: [react(), tailwindcss()]
       ```

    3. Update src/renderer/src/assets/main.css:
       ```css
       @import "tailwindcss";
       ```

    4. For shadcn/ui init to work, create vite.config.ts symlink or copy in project root pointing to electron.vite.config.ts renderer config. Alternative: manually create components.json and utils.ts.

    5. Run `npx shadcn@latest init` with these choices:
       - Style: Default
       - Base color: Slate
       - CSS variables: Yes
       - Path aliases configured: @ -> src/renderer/src

    6. If shadcn init fails due to framework detection, manually create:
       - components.json with paths pointing to src/renderer/src/components/ui
       - src/renderer/src/lib/utils.ts with cn() helper

    7. Ensure electron.vite.config.ts has path alias:
       ```typescript
       renderer: {
         resolve: {
           alias: {
             '@': resolve('src/renderer/src')
           }
         }
       }
       ```

    8. Update App.tsx to use Tailwind classes:
       ```tsx
       <div className="flex min-h-screen items-center justify-center bg-slate-900">
         <h1 className="text-4xl font-bold text-white">Focus</h1>
       </div>
       ```

    9. Test with `npm run dev` - should see styled heading.
  </action>
  <verify>
    npm run dev
    App shows centered white "Focus" text on dark background
    Tailwind classes are applied (not raw CSS)
  </verify>
  <done>Tailwind v4 and shadcn/ui configured, styling works</done>
</task>
